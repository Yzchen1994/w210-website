Our Products

Using multiple Machine Learning models to process NYC traffic and accident data, our team provides an accident prediction service that provides a list of potential accident locations along a given route from starting point to destination.

  • The web application could helps the drivers at NYC to pay more attention when driving close to the potential accident points.
  • Our API could also help the logistics companies to get the most frequent traffic accident points in NYC so they could optimize the routes of their fleets to avoid potential traffic accidents and reduce associated costs.

Datasets

We utilized a variety of datasets to train our accident prediction models.

Traffic accident datasets

A historical traffic accident dataset of 49 states of the United States from February 2016 to June 2020.

Weather Datasets

  • Historical weather information from NOAA
  • Real-time weather information from National Weather Service API

Traffic Datasets

  • Historical traffic information from public DOT data from NYC
  • Real-time traffic data for prediction was build based on the historical traffic

  • 01 Feature Selection

    According to feature importance analysis along with initial EDA and covariance matrix, four major feature group with 12 features was selected:

    • - Time
    • - Location
    • - Weather
    • - Traffic flow

  • 02 Generate Negative Samples

    Our accident dataset only contains positive data points. According to the Yuan et al., for every data point, we randomly changed the time and corresponding weather and traffic information. Those data were created based on a 3:1 negative/positive ratio.

  • 03 Build ML models

    We tested several algorithms to build our model and we found that the random forest model with categorical data achieves the highest accuracy and AUC. Therefore, we have chosen to use this model for our web application.

  • 04 Implementation of an interactive web application

    The front end will be similar to Google Map App with a simulation function to allow user to test the real-time driving experience. The web application is hosted on AWS and was developed using Flask, a python-based server-side language. For the interactive map application, the backend engine will:

    • - Call Google Map API to return a proposed route with a set of GPS coordinates along the route.
    • - Call Weather API for the real-time weather information with time and location.
    • - Call self-built Traffic Flow model for simulated real-time traffic information
    • - Feed all predictors from all three APIs into the prediction model and return a high-risk level of accidents along the route.
    • - Users can either simulate the driving experience or real driving testing with GPS locations. When getting close to the predicted high-risk level accident points, the web application will send a warning message to alter the user.
